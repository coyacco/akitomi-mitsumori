## 見積書管理システム

## プロジェクト概要
本プロジェクトは、見積書の作成・編集・管理を行う業務アプリケーションです。  
React（フロントエンド）と Rust（Axum / SQLite）を使用し、  
一覧表示、詳細表示、編集、PDF 出力など、見積書管理に必要な機能を提供します。

主な機能:
- 見積書の一覧・検索・詳細表示
- 見積書の新規作成・編集・削除
- 明細行の自動計算（数量 × 単価）
- 社員マスタとの連携（作成者のプルダウン選択）
- PDF 出力（将来的に実装予定）
- SQLite による軽量なデータ管理

# インストール手順（初めてご利用の方へ）

このセクションでは、見積書管理システムをセットアップする手順を説明します。

## システム構成

本システムはクライアント・サーバー型です。

- **サーバー**：見積書データを管理するサーバーアプリケーション（Windowsサービスとして動作）
- **クライアント**：見積書の作成・閲覧・編集を行うアプリケーション

## セットアップ手順

### ステップ 1: サーバーインストーラーをダウンロード

1. [Releases](https://github.com/coyacco/akitomi-mitsumori/releases) ページにアクセス
2. サーバーインストーラー（`AkitomiMitsumori-Server_X.X.X-setup.exe`）をダウンロード

### ステップ 2: サーバーをインストール

1. ダウンロードしたサーバーインストーラーを実行
2. 指示に従ってインストールを完了
3. インストール後、サーバーアプリケーションが **Windowsサービス** として自動的に起動します

### ステップ 3: サーバーのIPアドレスを確認

サーバーが起動しているか、またはIPアドレスを確認するため、以下の手順を実行してください。

#### Windowsサービスの確認

1. サーバーPC上で `Windows + R` キーを押して、「ファイル名を指定して実行」を開く
2. `services.msc` と入力して Enter キーを押す
3. サービス一覧から **akitomi-server**（またはそれに類する名前）を探す
4. **状態が「実行中」** であることを確認

#### IPアドレスの確認方法

1. サーバーPC上でコマンドプロンプト（`cmd.exe`）を開く
2. 以下のコマンドを実行：
   ```
   ipconfig
   ```
3. 出力から **IPv4 アドレス** を探す（例：`192.168.1.100`）

### ステップ 4: クライアントインストーラーをダウンロード

1. [Releases](https://github.com/coyacco/akitomi-mitsumori/releases) ページから、クライアントインストーラー（`AkitomiMitsumori_X.X.X_x64-setup.exe`）をダウンロード

### ステップ 5: クライアントをインストール

1. 見積書システムを利用したい **各PC** でクライアントインストーラーを実行
2. 指示に従ってインストールを完了

### ステップ 6: クライアントを起動してサーバーに接続

1. クライアントアプリケーションを起動
2. 初回起動時、サーバーIPアドレスの入力画面が表示されます
3. ステップ 3 で確認した **サーバーのIPアドレスを入力**（例：`192.168.1.100`）
4. 「接続」ボタンをクリック

### ステップ 7: 接続確認

- エラーメッセージが表示されず、見積書の一覧が表示されればセットアップ完了です
- 接続に失敗した場合は、以下の **トラブルシューティング** を参照してください

---

## トラブルシューティング

### 接続エラーが表示される場合

以下を確認してください：

1. **サーバーのIPアドレスが正しいか**
   - ステップ 3 で確認したIPアドレスを再度確認してください
   - タイプミスがないか確認してください

2. **Windowsサービスが起動しているか**
   - `services.msc` からakitomi-serverのサービスが **「実行中」** になっているか確認
   - 起動していない場合は、サービスを選択して「スタート」をクリック

3. **ファイアウォールの設定**
   - Windowsファイアウォールがサーバーのアプリケーション通信をブロックしていないか確認
   - 必要に応じてファイアウォール設定を調整してください

4. **ネットワークの確認**
   - サーバーPCとクライアントPCが同じネットワークに接続していることを確認
   - ネットワーク接続が正常に機能しているか確認

# 開発環境セットアップ

## 1. Rust / Axum API サーバーの起動手順（開発時）

```
cd akitomi-mitsumori-viewer\server
cargo build
.\target\debug\akitomi-server.exe
```

サーバーは以下で起動します：

```
http://localhost:3001
```

---

## 2. クライアント（Tauri + React）の起動手順

### 2-1. Tauri アプリとして起動（推奨）

```
cd akitomi-mitsumori-viewer\client
npm run tauri dev
```

Tauri ウィンドウが立ち上がり、アプリが動作します。

---

### 2-2. Vite（ブラウザ）で起動する場合

```
cd akitomi-mitsumori-viewer\client
npm run dev
```

ブラウザで以下へアクセス：

```
http://localhost:1420/
```

# API 一覧

## 見積書ヘッダ
- `GET /api/mitsumori/header/:no`  
- `POST /api/mitsumori/create`  
- `POST /api/mitsumori/update/:no`  
- `DELETE /api/mitsumori/:no`  

## 見積書明細
- `GET /api/mitsumori/detail/:no`

## 社員マスタ
- `GET /api/shain`  
- `POST /api/shain/add`（即時登録・採番）  
- `POST /api/shain/visible`（表示/非表示切替）
